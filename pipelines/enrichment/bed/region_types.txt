# naming convention for regions is:
#   regionClass/regionType == regionClass/spanType_scoreType_filtering
# where:
#   'regionClass' is used to group region BED files into folders, e.g. 'histone'
#   'regionType' == spanType_scoreType_filtering is standarized to reveal:
#       'spanType'  = the kind of span defined by each BED row, e.g., 'bins', 'TSS', 'peaks', etc.
#                     set spanType to 'spans' for spans that retain no inherent biological meaning or technical definition
#       'scoreType' = the nature of the score carried in BED score column 5, e.g., 'txn_rounds' for Pro-seq, 'fraction_gc', etc.
#       'filtering' = whether/how the spans of type spanType were filterd for inclusion in the BED file, e.g., 'active'
#                     set filtering to 'all' for unfiltered spans representing the whole genome, as used for score (not overlap) plots
# examples:
#   'genome/bins_fraction_gc_all' lists:
#        1kb genome bins
#        scored by bin fraction GC
#        unfiltered, i.,e. bins across the entire included genome are found in the regions BED file
#   'annotation/tss_txn_rounds_active' in class 'transcription' lists:
#        TSS regions
#        scored by Pro-seq txn in rounds
#        filtered to active TSS only

# inherent properties of the genome irrespective of measured data
